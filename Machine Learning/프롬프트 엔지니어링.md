# 프롬프트 엔지니어링 (Prompt Engineering)

---

> AI 모델에게 주는 지시문(프롬프트)을 설계하고 최적화하는 기술
> AI 언어 모델로부터 원하는 답변을 효과적으로 얻기 위한 기술이다

- 같은 질문이라도 어떻게 물어보느냐에 따라 답변의 질이 달라짐
- ChatGPT, Claude, GPT-4 등 대규모 언어 모델(LLM) 활용의 핵심
- 개발자, 연구자, 일반 사용자 모두에게 중요한 스킬

## 왜 프롬프트 엔지니어링이 필요한가?

**단순한 질문의 문제점**

- 모호한 질문은 모호한 답변을 받음
- 맥락이 부족하면 원하는 결과를 얻기 어려움
- AI가 의도를 잘못 이해할 수 있음

**프롬프트 엔지니어링의 장점**

- 정확하고 구체적인 답변 획득
- 작업 효율성 크게 향상
- AI의 능력을 최대한 활용
- 일관성 있는 결과물 생성

## 핵심 원칙

### 1. 명확성 (Clarity)

**구체적이고 명확하게 지시하기**

- 나쁜 예: "코드 좀 작성해줘"
- 좋은 예: "Python으로 두 숫자를 더하는 함수를 작성해줘. 함수명은 add_numbers이고, 두 개의 정수를 받아서 합을 반환해야 해."

### 2. 맥락 제공 (Context)

**배경 정보와 상황 설명하기**

- 나쁜 예: "이메일 작성해줘"
- 좋은 예: "고객에게 제품 배송 지연을 알리는 공식 이메일을 작성해줘. 사과의 말과 함께 예상 배송일을 포함해야 하고, 친절하지만 전문적인 톤을 유지해줘."

### 3. 역할 부여 (Role Assignment)

**AI에게 특정 역할이나 관점 부여하기**

```
"당신은 10년 경력의 소프트웨어 아키텍트입니다.
마이크로서비스 아키텍처의 장단점을 설명해주세요."
```

위 예시와 같이 역할을 부여하면 답변의 퀄리티가 올라간다.

### 4. 단계별 지시 (Step-by-Step Instructions)

**복잡한 작업은 단계별로 나누기**

```
다음 순서대로 작업해줘:
1. 먼저 문제를 분석해줘
2. 가능한 해결책 3가지를 제시해줘
3. 각 해결책의 장단점을 비교해줘
4. 최종 추천안을 선택하고 이유를 설명해줘
```

LLM에게 한꺼번에 많은 요구를 하기보다 단계별로 순차적으로 처리를 요청하는 것이 좋은 결과를 얻기 쉽다.

## 주요 기법

### Few-Shot Learning

**예시를 제공해서 원하는 형식 보여주기**

```
다음 예시처럼 제품 설명을 작성해줘:

입력: 무선 마우스
출력: 편안한 그립감의 인체공학적 무선 마우스.
배터리 수명 6개월, 정밀한 센서로 작업 효율 향상.

입력: 블루투스 스피커
출력: [AI가 같은 형식으로 작성]
```

예시를 제공하는 것이 답변을 의도대로 작성 할 수 있다.

### Chain-of-Thought (CoT)

**사고 과정을 단계별로 진행하도록 유도**

```
이 문제를 단계별로 생각해보자:
1. 먼저 주어진 정보를 정리하면...
2. 그 다음 고려해야 할 요소는...
3. 따라서 결론은...
```

LLM의 사고 과정을 단계별로 작성하면 도움이 된다.

### Zero-Shot Prompting

**예시 없이 직접적인 지시만으로 작업 수행**

```
"다음 텍스트의 감정을 '긍정', '부정', '중립' 중 하나로 분류해줘."
```

### 제약 조건 명시

**출력 형식이나 길이 등을 명확히 지정**

```
"200자 이내로 요약해줘."
"JSON 형식으로 출력해줘."
"불릿 포인트 5개로 정리해줘."
```

제약 조건을 지정하면 원치않는 답변을 방지 할 수 있다.

## 고급 테크닉

### 1. 시스템 프롬프트 활용

```
System: 당신은 도움이 되고, 정직하며, 무해한 AI 어시스턴트입니다.
User: [질문]
```

### 2. 반복적 개선

- 초기 프롬프트로 결과 확인
- 부족한 부분 파악
- 프롬프트 수정 및 재시도
- 최적의 결과를 얻을 때까지 반복

### 3. 템플릿화

**자주 사용하는 프롬프트를 템플릿으로 저장**

```
[역할]: 당신은 {전문 분야} 전문가입니다.
[작업]: {구체적 작업}을 수행해주세요.
[제약조건]:
- {조건1}
- {조건2}
[출력형식]: {원하는 형식}
```

### 4. 프롬프트 체이닝

**여러 단계의 프롬프트를 연결해서 복잡한 작업 수행**

```
Step 1: "이 코드를 분석해줘"
Step 2: (분석 결과를 받은 후) "이제 이 분석을 바탕으로 개선안을 제시해줘"
Step 3: (개선안을 받은 후) "제시된 개선안을 코드로 구현해줘"
```

## 프롬프트 엔지니어링 체크리스트

실제 사용 전 확인사항:

- [ ] 요청이 구체적이고 명확한가?
- [ ] 필요한 맥락 정보를 제공했는가?
- [ ] 원하는 출력 형식을 명시했는가?
- [ ] 제약 조건이나 요구사항을 빠짐없이 전달했는가?
- [ ] 예시가 필요하다면 제공했는가?
- [ ] 복잡한 작업을 단계별로 나눴는가?
